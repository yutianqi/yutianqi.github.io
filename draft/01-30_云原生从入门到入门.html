云原生从入门到入门

没错，就是从入门到入门。这里只有你需要了解的尽量少的知识。

开发环境

请停下你正在卡带上打孔的双手，你需要现代的工具：

  • 电脑(Win11/Mac/Linux)
  • JDK，1.8
  • Maven 3.8.6
  • IDEA 2023.2
  • Spring版本
  • MobaXterm/XShell
  • FileZilla/XFtp
  • 华为云

先要有个Web应用

你要用云来运行什么呢，不如创建一个提供RESTFul接口的Spring应用

  1. 生成Spring项目框架代码
  2. 下载项目代码并导入IDEA
  3. 开发Rest接口
  4. 在IDEA中运行，并在浏览器中访问

至此，你已经开发完了一个功能强大的Spring应用，但是你不会希望你的项目单点部署在一台需要安装IDEA的个人电脑上。
所以你的项目需要具备独立部署的能力，这时聪明的你决定将项目打包成jar文件。

生成并运行你的jar文件

  1. 将源码打包成可运行的jar包
  2. 独立运行jar包 至此，你可以手动分发你的jar文件。在任意一台安装的jdk或者jre的机器上，都可以通过java命令运行你的Spring应用。
    但是就如前面所说，你的机器需要安装jdk或者jre，并且不同的人可能会将你的jar文件丢到不同的目录，使用不同的命令来运行。
    作为完美主义者的你当然是不允许的，所以你打算使用docker部署。

Docker部署

  1. 为了打包，你最好有一台Linux服务器，这时你可以登录Linux服务器后台
  2. 安装好Docker程序
  3. 此外因为网络的原因，你最好把镜像源设置为国内的镜像源
  4. 创建一个你能记住的临时目录，比如/opt/docker_test
  5. 使用FileZilla或者XFtp等工具，上传前面生成的jar文件Spring-0.0.1-SNAPSHOT.jar到/opt/docker_test目录
  6. 创建并编写Docker文件来告诉Docker等会如何构建你的镜像
  7. 让Docker按照你的要求构建镜像
  8. 查看构建出的镜像
  9. 运行你的镜像
  10. 查看镜像运行的结果：容器
  11. 进入容器
  12. 将你的镜像推送到镜像仓库

至此，你已经成功使用最流行的Docker技术来部署和分发你的应用。
每个有权限获取你镜像的人，都可以在任意一台安装了Docker的机器上，将你的镜像部署成容器，来体验你创造的伟大应用并感叹这个世界的美好。
但是如果运行镜像的机器重启了，或者是用户太多，想部署更多实例。那他们还是将无法再体验你的伟大应用。你忍心直面那些人流露出失望的眼神吗？
所以你咬了咬牙，决定再进一步，使用k8s来管理容器。

        # 推送到阿里云
            https://baijiahao.baidu.com/s?id=1751699245331439817&wfr=spider&for=pc

            1. 登录阿里云Docker Registry
            $ docker login --username=3521*****@qq.com registry.cn-hangzhou.aliyuncs.com
            用于登录的用户名为阿里云账号全名，密码为开通服务时设置的密码。

            您可以在访问凭证页面修改凭证密码。

            2. 从Registry中拉取镜像
            $ docker pull registry.cn-hangzhou.aliyuncs.com/dpframework/testrepo:[镜像版本号]

            3. 将镜像推送到Registry
            $ docker login --username=3521*****@qq.com registry.cn-hangzhou.aliyuncs.com
            $ docker tag [ImageId] registry.cn-hangzhou.aliyuncs.com/dpframework/testrepo:[镜像版本号]
            $ docker push registry.cn-hangzhou.aliyuncs.com/dpframework/testrepo:[镜像版本号]
            请根据实际镜像信息替换示例中的[ImageId]和[镜像版本号]参数。

            4. 选择合适的镜像仓库地址
            从ECS推送镜像时，可以选择使用镜像仓库内网地址。推送速度将得到提升并且将不会损耗您的公网流量。

            如果您使用的机器位于VPC网络，请使用 registry-vpc.cn-hangzhou.aliyuncs.com 作为Registry的域名登录。

            5. 示例
            使用"docker tag"命令重命名镜像，并将它通过专有网络地址推送至Registry。

            $ docker images
            REPOSITORY                                                         TAG                 IMAGE ID            CREATED             VIRTUAL SIZE
            registry.aliyuncs.com/acs/agent                                    0.7-dfb6816         37bb9c63c8b2        7 days ago          37.89 MB

            $ docker tag 37bb9c63c8b2 registry-vpc.cn-hangzhou.aliyuncs.com/acs/agent:0.7-dfb6816
            使用 "docker push" 命令将该镜像推送至远程。

            $ docker push registry-vpc.cn-hangzhou.aliyuncs.com/acs/agent:0.7-dfb6816
